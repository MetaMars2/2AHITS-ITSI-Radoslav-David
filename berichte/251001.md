# Grundlagen

Shell: ist ein Programm das eine textorientierte Schnittstelle zum Betriebssystem zur Verfügung stellt.

Prompt:

```sh
~/workspace$
```

Einfacher Befehl:

```sh
$ date
```

Jeder befehl hat Optionen über die das Verhalten gesteurt werden kann

```sh
$ date -I
```

## Optionen:
- short-and = 1 Buchstabe
- long-hand = Wort

```sh
$ date --iso-8601
```

## Argumente
Daten mit denen ein Befehl arbeitet

```sh
$ echo Hallo Welt
```

`Hallo Welt` ist hier das Argument

mit Optionen

```sh
$ echo -n Hallo Welt
```

## Tastenkürzel

- Pfeil nach oben/unten: Historie
- Tabulator: automatische File Vervollschtändigung
- Strg-A: Anfang der Zeile
- Strg-C: Abbruch des aktuellen Kommandos

## File Befehle
cd, mkdir, ls, touch, pwd, rm, rmdir

### cd
Mit `cd` wechselst du von deinem aktuellen Verzeichnis in ein anderes

### mkdir
Dieser Befehl erstellt ein neues, leeres Verzeichnis

### touch 
Der `touch`-Befehl wird hauptsächlich verwendet, um eine neue, leere Datei zu erstellen

### ls
Der `ls`-Befehl listet alle Dateien und Unterverzeichnisse im aktuellen oder einem angegebenen Verzeichnis auf

Argument `-l` (long format): Zeigt eine detaillierte Liste an
```sh
$ ls -l
```

Ausgabe
```sh
~/workspace$ ls -l
total 8
drwxr-xr-x 1 runner runner  42 Oct  1 07:25 berichte
-rw-r--r-- 1 runner runner 107 Sep 12 14:15 README.md
```

Argument `-a` (all): Zeigt alle Dateien an, auch versteckte, die mit einem Punkt . beginnen

```sh
~/workspace$ ls -a
.   berichte  .git    README.md  .upm
..  .cache    .local  .replit
```



### pwd
Das `pwd` Befehl zeigt dir den vollständigen Pfad des Verzeichnisses an

### rm
`rm` wird verwendet, um Dateien zu löschen

Argument `-r` (recursive): Löscht ein Verzeichnis und seinen gesamten Inhalt (alle Unterordner und Dateien darin

```sh
rm -r
```

### rmdir
Das `rmdir` Befehl löscht leere Verzeichnisse


Unteschied absoluter und relativer Pfad

Bedeutung von cd (ohne Argumente) und von ~
