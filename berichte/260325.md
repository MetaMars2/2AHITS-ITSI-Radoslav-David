### 1. Programm starten

```bash
chmod +x pwcheck
./pwcheck
```

Das Programm wird ausführbar gemacht und gestartet. In der Konsole erscheint eine Passwortabfrage.

---

### 2. Binärdatei analysieren

```bash
xxd pwcheck | more
```

Die Datei wird als Hexdump angezeigt. Man sieht:

* links die Offsets (Adressen)
* in der Mitte die Hexwerte
* rechts die ASCII-Darstellung

So erkennt man die Struktur der Binärdatei.

---

### 3. Analyse mit gdb

```bash
gdb pwcheck
b main
run
disas main
```

Das Programm wird im Debugger gestartet.

* `b main` setzt einen Breakpoint
* `run` startet das Programm
* `disas main` zeigt den Assemblercode

Man kann den Ablauf des Programms Schritt für Schritt beobachten.

---

### 4. Passwortprüfung verstehen

Im Assemblercode findet man:

```
cmp $0x0,%eax
je 1263
```

Das Ergebnis der Passwortprüfung wird verglichen.
Der Befehl `je` sorgt dafür, dass bei falschem Passwort gesprungen wird.

---

### 5. Manipulation im gdb

```bash
set {unsigned char}0x11c8 = 0x90
```

Der Sprungbefehl wird durch NOP (0x90) ersetzt.
Dadurch wird die Passwortprüfung übersprungen und jedes Passwort akzeptiert.
